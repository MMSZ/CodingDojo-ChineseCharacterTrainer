Entity Framework quick start

by Andy R • September 13, 2011 • 0 Comments

Microsoft’s Entity Framework has been around a while, but with it’s latest incarnation it is really shaping up nicely. I’ve been enjoying the ‘code first’ approach in version 4.1 of the framework. This allows you start your application design from POCOs, allowing the framework to infer the database design for you. For example, take the following Member entity model:

view sourceprint?
1.class Member
2.{
3.    public int Id { get; set; }
4.    public string FirstName { get; set; }
5.    public string LastName { get; set; }
6.}


The following class derives from the DbContext base class, and exposes our Member model:

view sourceprint?
01.class EFDbContext : DbContext
02.{
03.    public DbSet<Member> Members { get; set; }
04. 
05.    public EFDbContext()
06.    {
07.        Database.SetInitializer<EFDbContext>(new DbContextInitializer());
08.    }
09.}

Note the SetInitializer call in the constructor. This allows us to seed the database with some data, using the following class:

view sourceprint?
01.class DbContextInitializer : DropCreateDatabaseIfModelChanges<EFDbContext>
02.{
03.    protected override void Seed(EFDbContext Context)
04.    {
05.        new List<Member>
06.        {
07.            new Member {
08.                FirstName = "Andrew",
09.                LastName = "Rowland"
10.            },
11.            new Member {
12.                FirstName = "Roland",
13.                LastName = "Browning"
14.            },
15.        }.ForEach(m => Context.Members.Add(m));
16.         
17.        base.Seed(Context);
18.    }
19.}

Now we need a simple repository class, so we can query the database:

view sourceprint?
01.class MemberRepository
02.{
03.    private EFDbContext Context = new EFDbContext();
04. 
05.    public IQueryable<Member> Members
06.    {
07.        get { return Context.Members; }
08.    }
09.}

Standard practice is to use an interface, and obtain this concrete class using an IOC container, but it will do for demonstration purposes.

Due to lazy loading, nothing would happen until we actually run a query. So finally, a console application to instantiate the repository, and display the data:

view sourceprint?
01.class ConsoleApplication
02.{
03.    static void Main(string[] args)
04.    {
05.        MemberRepository Repository = new MemberRepository();
06. 
07.        Repository.Members.ToList().ForEach(m => Console.WriteLine(String.Format("Member: {0} {1}", m.FirstName, m.LastName)));
08. 
09.        Console.ReadKey();
10.    }
11.}

We can place all of the code into one console application. Install EF4.1, create a console application in Visual Studio 2010, paste in the class below. You may have to set the startup object in the project properties.

view sourceprint?
01.using System;
02.using System.Collections.Generic;
03.using System.Linq;
04.using System.Text;
05.using System.Data.Entity;
06. 
07.namespace EFExample
08.{
09.    class Member
10.    {
11.        public int Id { get; set; }
12.        public string FirstName { get; set; }
13.        public string LastName { get; set; }
14.    }
15. 
16.    class EFDbContext : DbContext
17.    {
18.        public DbSet<Member> Members { get; set; }
19. 
20.        public EFDbContext()
21.        {
22.            Database.SetInitializer<EFDbContext>(new DbContextInitializer());
23.        }
24.    }
25. 
26.    class DbContextInitializer : DropCreateDatabaseIfModelChanges<EFDbContext>
27.    {
28.        protected override void Seed(EFDbContext Context)
29.        {
30.            new List<Member>
31.            {
32.                new Member {
33.                    FirstName = "Andrew",
34.                    LastName = "Rowland"
35.                },
36.                new Member {
37.                    FirstName = "Roland",
38.                    LastName = "Browning"
39.                },
40.            }.ForEach(m => Context.Members.Add(m));
41.             
42.            base.Seed(Context);
43.        }
44.    }
45. 
46.    class MemberRepository
47.    {
48.        private EFDbContext Context = new EFDbContext();
49. 
50.        public IQueryable<Member> Members
51.        {
52.            get { return Context.Members; }
53.        }
54.    }
55. 
56.    class ConsoleApplication
57.    {
58.        static void Main(string[] args)
59.        {
60.            MemberRepository Repository = new MemberRepository();
61. 
62.            Repository.Members.ToList().ForEach(m => Console.WriteLine(String.Format("Member: {0} {1}", m.FirstName, m.LastName)));
63. 
64.            Console.ReadKey();
65.        }
66.    }
67.}

Run the application, and the magic begins. If you have an instance of SQL Server Express running, a database with a single Members table will be created, populated with data, and the results displayed.
